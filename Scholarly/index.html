Summary

StoW (Stories on the Web) is a web application that offers children of all ages a friendly graphical interface for reading their favourite stories. The users will be able to search for their favorite story on the site (with or without login) and will be able to read and interact with the storyline in a nice way.
By logging in, the users will be able to upload their own stories and use features such as: leaving a bookmark, adding the story to favourites, rating stories and discussing them with other users. The application will respect the principles of responsive design.

Introduction

Motivation

  • Offering a pleasant and interactive narrative experience via an user-friendly interface.
  • Offering a way of storing and managing an ever-growing collection of stories.
  • Offering our users the possibility of adding new stories and expanding our universe.
  • Offering our users stories tailored to their own needs, based on their age and preferred genres.

StoW is destined for children, making them our main demographic, but their parents are also a target audience of our application.

How to use the application: After entering the web page's URL in the browser's adress bar, there are multiple options:

The Menu

  • The "Home" button in the upper left side will take you to the homepage of our app.
  • Right next to it, the "Browse" button will take you to the browse view.
  • The "Upload" button allows an user to upload new stories (in an XML format). This feature will require the user to log in.
  • You can search for a specific story by entering its name in the "Search" bar. This will also take you to the browse view.
  • The "Sign in" button on the upper right will take the user to a sign in/register view.

The Homepage View:

  • This is the view that the user will face when first starting the application.
  • The user will be greeted with a colourful banner, with two buttons - upload and browse. Those buttons can also be found in the menu on the top, among other tools useful for navigating the website.
  • Below the banner are listed a few of the most recently added stories.

The Browse View:

  • This is the interface that allows the user to navigate through our stories, and find something worth a read.
  • The user can filter the stories by age or by genre. Or use the search function to find something specific.
  • The resulting array of stories can be navigated via the "next" and "previous" buttons.
  • On hovering, the covers of the displayed stories will reveal more information about them, such as title and user rating.
  • To open a story the user has to click on the cover of the prefered book.

The Story View:

  • Here, the user can read the story he/she wants. By clicking next, the next page of the story will be loaded.
  • If the story is interactive, instead of a next button, there will be a list of options. The reader then has to decide how the story continues.
  • At the bottom, the progress bar is shown, indicating how much the user has already read.
  • The "A+" and "A-" buttons will zoom the text in and out.
  • A bookmark can be placed on the page by clicking the icon on the left side, so that the next time the user logs in, he/she can continue from where he/she left off.
  • The story can be rated with 1 to 5 stars and can be added to favorites by clicking the "heart" icon.

The backend

  • Compiled a list of features that the final product should have (use cases).
  • Sketched the database architecture: The content of the stories will be structured in a linked list of "blocks" (where a block could have links to multiple other blocks - to support interaction with the reader) which will be stored in a separate table of the database.

THE DATABASE

Structure of the database:

  • Stories (story_id (pk), title, author, content(link to XML file))
  • Categories (category_id (pk), type, name)
  • StoriesXCategories ( story_id (pk, fk), category_id (pk,fk))
  • Characters (character_id (pk), story_id(pk,fk), character(link to XML file))
  • Users (user_id (pk), name, surname, username, password, e-mail, etc.)
    • Stores all users, regardless of type.
  • Roles (role_id (pk), role_name i.e. admin, or other special user types)
  • User_Roles (user_id (fk, pk), role_id (fk, pk))
    • Assigns roles to special users.
  • Bookmarks (user_id (fk, pk), story_id (fk, pk), page_id) )
  • Favourites (user_id (fk, pk), story_id (fk, pk))
  • Comments (id (pk), user_id (fk), story_id (fk), comment)

ACTUAL IMPLEMENTATION OF STOW

On the home page you can see: "Browse", "Upload", "Search", "Account" buttons and the recently added stories.

  • Tehnical details:

    • In the home page, in order to populate the most recently added 5 stories we make a request to the server that will query the database and will return the title, rating, authors and the json in which every story keeps its content (paths to images and text), in order to populate the thumbnails for every story.

  • Front-end:

    • The whole page has a maximum width of the content of 960px to ensure good reading on monitors of any size without straining the eyes. Resizing the window to a smaller size will not make the content unreadable. There is also a mobile version that "activates" when the browser window has a width of 500px or lower. In order to make this website flexible, flex-box was used. The menu is fixed and always will be present at the top of the page, no matter how far down you scroll.

    The mobile version is very similar to the desktop version and it's easy to navigate through it.

    When you press "Browse", you will be presented with a page with filters: "AGE GROUP", "GENRE". You can also order yoUr results after applying filters by seleting either "RATING", "DATE ADDED" or "ALPHABETIC" from the drop down lists. Ordering can be either ascending or descending. On the bottom of the page you can see the pagination section with the buttons: "FIRST", "PREV" ,"NEXT" and "LAST".

  • Tehnical details:

    • In order to populate with the desired stories, a POST is made to the server with all the filters necessary, or none if the user intended so. When you choose a new filter or want a different ordering or press the next or prev button an AJAX request is made, in order not to reload the page and give a better user experience.You can choose as many filters as you like, they will never duplicate. The server will execute the "Browse.php" controller and give back the result returned by the find function from the "Story.php" model.

    When you are in the Filters area and you select a category for example, that category will go in the Filtered results area. You may remove your filters by clicking on them (they have a small 'x' sign to indicate they can be removed).

    When you press the “Upload”, button you will get an alert message “You need to be signed in to access this feature. Please sign in or sign up below!”, and the you will see the “SIGN IN” “SIGN UP” buttons (only when yOu aren't logged in).

  • Tehnical details:

    • If the $_SESSION["userData"] variable is not set the user is redirected to this page in which he has to sign in or sign up.

    When you press "Sign In", you will see a form with the message "Welcome Back!:)", the "Username" and "Password" fields and the "Submit" button.

  • Tehnical details:

    • After pressing the Submit button a query is sent to the database with the User Name and the Password (which is encoded in md5). If the result is not null then the user will be redirected to the Home page or, if the $_SESSION["redirectPage"] is set, to the latest story that he was reading.

    When you press "Sign Up", you will see a form with the message "Join the fun! With an account, you can add bookmarks, ratings, comments and more! ", the "Username", "Password", "Pepeat password", "Email", "Name", "Surname" fields and the "Submit" button. While you fill the required data, you will get feedback in real time.

  • Tehnical details:

    • Again, after pressing the Submit button the data from the form will be collected and an AJAX post will be made to the server. If all is ok, the server will insert the information of the user into the database, sign them in and redirect them to the home page.

    When you press on the "Search" button, you will see a field where you can type a story title. Hitting enter or clicking the search button will redirect you to the Browse page.

    When you hover over "Account", you will see the messages "Sign In" and "Sign Up".
